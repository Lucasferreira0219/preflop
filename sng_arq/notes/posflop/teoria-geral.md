# Pós-Flop — Teoria e Guidelines Gerais

## Resumo executivo
- Aula simplifica pós-flop para aplicação em limites baixos.
- Foco em **3 situações frequentes**:
  1. IP vs BB (abrimos, só BB defendeu) — **mais comum**.
  2. OOP vs jogador em posição.
  3. 3-bet pots IP.
- Estratégia central: **c-bet 100%, size 1/3** na situação 1.

## Matemática da c-bet
- Tamanho do bet define **fold equity necessária**.
- 50% pot → vilão precisa foldar **33%** das vezes para ser lucrativo.
- Quanto menor a aposta, menor a frequência necessária de fold.

## Guidelines de c-bet

### IP vs BB
- **C-bet 100%, size 1/3 do pote**.
- Regra: "fecha o olho e aposta 33%".
- Não existe motivo pra dar check.

### OOP vs jogador IP
- **C-bet menos frequentemente**, sizes maiores (1/2 a 2/3).
- Priorizar mãos com equidade (não bluff catcher puro).

### 3-bet pots IP
- **C-bet 100%, size 1/3** (mesmo padrão).

## Jogando como caller (foi quem pagou pré)
- Vilão checka → aposta ~100%, size 1/3.
- Vilão c-betou → continuar com pares, draws, equidade.

## Pós-flop após punir limp
- C-bet 100%, size **25-33%**.
- **Evitar slow play** com mãos fortes — sempre construir pote.
- Evitar blefes exagerados — recreativos pagam muito.

## Respeitando agressão
- Field passivo + vilão aposta forte múltiplas streets = mão forte.
- **Não atolar** com mãos médias — preserva stack pra fases futuras.

## Conceitos-chave
- **C-bet 1/3 pote** é o tamanho-âncora.
- **IP vs BB é a situação 1** (a mais frequente).
- Recreativos no field não defendem o suficiente → small bets são profitáveis.
- Mentalidade: **simples e disciplinada**, não complexa.

## Heurísticas pro app (futuro distante)
- App hoje é preflop-only. Pós-flop seria expansão maior.
- Possível mini-módulo educacional explicando essas guidelines.
- Não vira range diretamente.
